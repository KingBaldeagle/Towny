# Economy

## Providers
NeoTowny supports two economy providers configured via `Config.ECONOMY_PROVIDER`:
- **`lightmans_currency`** ‚Äì Integrates with the LightmansCurrency mod. Uses the team system to create accounts for players, towns and nations. This is the default and provides full functionality.
- **`closed`** ‚Äì A no‚Äëop provider that disables all economic actions. Useful for creative worlds or when no economy mod is installed.

## Account IDs
All accounts are identified by a unique string generated by `TownyEconomyHandler`:
- **Player** ‚Äì `player_<UUID>` (internally just the UUID string).
- **Town** ‚Äì `town_<townName>` where `<townName>` is the town‚Äôs name.
- **Nation** ‚Äì `nation_<nationName>`.
- **Server** ‚Äì A team named `server` used for towns that are not part of a nation.

## Money Flow
```
Player Wallet ‚Üí Town Account ‚Üí (optional) Nation Account ‚Üí Server Account
```
- When a player creates a town or nation, the required copper is withdrawn from their wallet and deposited into the newly created team account.
- Resident taxes move copper from the resident‚Äôs personal account to their town‚Äôs account.
- If a town belongs to a nation, a percentage of the town‚Äôs balance (`Config.NATION_TAX_PERCENT`) is transferred to the nation‚Äôs account each tax cycle.
- Towns without a nation send their surplus to the server team account.

## Tax Collection
- **Resident Tax** ‚Äì Executed every `Config.TAX_COLLECTION_INTERVAL_TICKS` ticks (default 24‚ÄØ000 ticks ‚âà‚ÄØ20‚ÄØminutes). Handled by `TownyEconomyService.collectResidentTax`. Residents that cannot pay are marked **delinquent**.
- **Nation Tax** ‚Äì After resident taxes, `TownyEconomyService.collectNationTaxes` distributes a configurable percentage of each town‚Äôs balance to its nation.
- **Delinquent Residents** ‚Äì Stored in a concurrent set. Delinquent residents cannot purchase plots until they pay their overdue taxes.

## Plot Purchases
When a plot is for sale and a resident runs `/plot claim`, the purchase flow (`TownyEconomyService.purchasePlot`) transfers the plot price from the buyer‚Äôs wallet to the owning town‚Äôs account, then updates plot ownership.

## Transaction Log
- Up to **500** recent transactions are kept in a synchronized list of `TownyTransactionRecord`.
- Each entry records:
  - Type (`resident_tax`, `nation_tax`, `plot_purchase`)
  - Source account, destination account
  - Amount (copper)
  - Success flag
  - Timestamp
  - Optional details (e.g., plot name)
- The log is cleared on server restart.

## Configurable Values (`Config.java`)
| Setting | Description |
|---|---|
| `PRICE_NEW_TOWN_IN_COPPER` | Cost to create a town. |
| `PRICE_NEW_NATION_IN_COPPER` | Cost to create a nation. |
| `TAX_COLLECTION_INTERVAL_TICKS` | How often resident taxes are collected. |
| `NATION_TAX_PERCENT` | Percentage of town balance sent to the nation each cycle. |
| `ECONOMY_PROVIDER` | Which provider to use (`lightmans_currency` or `closed`). |

## Runtime Inspection
Players and admins can view a summary via `/towny economy`, which displays:
- Active provider name
- Tax interval and nation tax percent
- Number of delinquent residents
- Transaction log entry count
- Sample currency formatting
